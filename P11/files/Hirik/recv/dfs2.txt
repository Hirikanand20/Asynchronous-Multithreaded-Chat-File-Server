## 1️⃣ Internal nodes vs External nodes

**Statement:**

> A binary tree with **n internal nodes** has exactly **n + 1 external nodes**.

### What does this mean?

* **Internal node** → a node that has **at least one child**
* **External node (leaf)** → a node with **no children**

### Example:

```
        A
       / \
      B   C
         /
        D
```

* Internal nodes: `A, C` → **n = 2**
* External (leaf) nodes: `B, D, (null right of C?)`

In a **proper counting model**, null children are considered external →
So external nodes = **n + 1 = 3**

✔️ Property holds.

---

## 2️⃣ Relation between leaf nodes and degree-2 nodes

**Statement:**

> For any non-empty binary tree,
> **n₀ = n₂ + 1**

Where:

* `n₀` = number of **leaf nodes**
* `n₂` = number of nodes with **degree 2** (two children)

### Example:

```
        A
       / \
      B   C
         / \
        D   E
```

* Leaf nodes (`n₀`): `B, D, E` → **3**
* Degree-2 nodes (`n₂`): `A, C` → **2**

✔️ `n₀ = n₂ + 1 → 3 = 2 + 1`

---

## 3️⃣ Maximum nodes at level *i*

**Statement:**

> Maximum number of nodes at level `i` is **2ⁱ**, where `i ≥ 0`

(Level counting starts from **0**)

### Example:

```
Level 0:        A          → 1 = 2⁰
Level 1:      B   C        → 2 = 2¹
Level 2:    D  E F  G      → 4 = 2²
```

So:

* Level 0 → 1 node
* Level 1 → 2 nodes
* Level 2 → 4 nodes

✔️ Matches **2ⁱ**

---

## 4️⃣ Maximum nodes in a binary tree of height *k*

**Statement:**

> Maximum nodes = **2ᵏ⁺¹ − 1**

This happens when the tree is **perfect** (completely filled).

### Example:

Height `k = 2`

```
        A
       / \
      B   C
     / \ / \
    D  E F  G
```

Nodes = `7`
Formula:

```
2^(2+1) − 1 = 2³ − 1 = 7
```

✔️ Correct.

---

## 5️⃣ Minimum and maximum height for *n* nodes

**Statement:**

> Height of a binary tree with `n` nodes is:

* **At least** ⌈log₂(n + 1) − 1⌉
* **At most** `n − 1`

### Minimum height (best case → balanced tree)

Example: `n = 7`

```
log₂(8) − 1 = 3 − 1 = 2
```

Height = **2** (perfect tree)

### Maximum height (worst case → skewed tree)

```
A
 \
  B
   \
    C
     \
      D
```

For `n = 4` → height = **3 = n − 1**

✔️ Both bounds make sense.

**Statement:**

This is the **Catalan Number**.

For `n = 3`:

```
## ✅ Inorder Traversal Using Stack (Pseudocode)

**Inorder = Left → Root → Right**

### Pseudocode:

```
INORDER_TRAVERSAL(root)
{
    Stack S
    Node current = root

    while (current != NULL OR S is not empty)
    {
        // Step 3: Reach the leftmost node
        while (current != NULL)
        {
            S.push(current)
            current = current.left
        }

        // Step 4: Current is NULL, process stack
        current = S.pop()
        print current.data

        // Step 4(c): Visit right subtree
        current = current.right
    }
}

Node* getSuccessor(Node* curr) {
    curr = curr->right;
    while (curr != nullptr && curr->left != nullptr)
        curr = curr->left;
    return curr;
}

// Delete a node with value x from BST
Node* delNode(Node* root, int x) {
    if (root == nullptr)
        return root;

    if (root->data > x)
        root->left = delNode(root->left, x);
    else if (root->data < x)
        root->right = delNode(root->right, x);
    else {
        // Node with 0 or 1 child
        if (root->left == nullptr) {
            Node* temp = root->right;
            delete root;
            return temp;
        }
        if (root->right == nullptr) {
            Node* temp = root->left;
            delete root;
            return temp;
        }

        // Node with 2 children
        Node* succ = getSuccessor(root);
        root->data = succ->data;
        root->right = delNode(root->right, succ->data);
    }
    return root;
}

## 🔥 MOST DIFFICULT BST CASE

* **2 children**
* **Successor is NOT a leaf**
* **Successor has a right child**
* **Deletion happens deep in tree**
